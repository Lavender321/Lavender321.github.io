1、对BFC的理解

创建方式：

根元素（例如）或包含它的元素。
浮动元素（元素的float不是none）。
绝对定位元素（元素的position为absolute或fixed）。
内联块（元素的display为inline-block）。
表格单元格（元素的display为table-cell，HTML表格单元格默认值）。
表格标题（元素的display为table-caption，HTML表格标题默认值）。
匿名表格单元格（元素的display为table、table-row、table-row-group、table-header-group、table-footer-group（以及inline-table、table-cell、table-column、table-column-group）。
overflow 值不为 visible 的块元素（overflow为auto、scroll、hidden）。
flex 或 grid 容器（元素的display为flex 或 inline-flex、grid 或 inline-grid）。
BFC的特性和用途
BFC有一些重要的特性和用途，包括但不限于：

内部的Box会在垂直方向上一个接一个的放置。
Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠。
BFC的区域不会与float元素的Box重叠。可以用来实现自适应两栏布局。
计算BFC的高度时，浮动元素也参与计算。这可以解决父元素由于子元素浮动导致内部高度为0的问题。
BFC的应用场景
防止margin重叠：对于属于同一个BFC的两个相邻Box，其margin可能会重叠。通过创建新的BFC可以避免这种情况。
布局设计：利用BFC可以实现多栏布局，例如侧边栏与内容区域并排布局。
清除浮动：创建BFC可以包含浮动元素，防止父容器高度塌陷。
理解BFC对于深入理解CSS布局非常重要，能够帮助解决许多布局问题，提高页面渲染的灵活性和稳定性。

2、浏览器的事件循环

浏览器的事件循环（Event Loop）是一个在浏览器中运行的过程，用于处理异步事件。它允许JavaScript在单线程环境中运行非阻塞（异步）操作，如处理I/O、定时事件（setTimeout、setInterval）和UI事件等。

事件循环的工作机制可分为以下几个关键部分：

1. 调用栈（Call Stack）

  • JavaScript代码执行时，所有的函数调用都会被压入到调用栈中。当函数执行完毕，它们会从栈中弹出。

2. 事件队列（Event Queue）

  • 当异步事件发生时（例如，用户点击、网络请求返回等），相应的回调函数会被添加到事件队列中。如果调用栈为空，事件循环会从事件队列中取出任务执行。

3. 微任务队列（Microtask Queue）

  • 微任务（例如Promise的回调）也是异步任务，但它们的优先级高于常规的事件队列任务（宏任务）。每次调用栈清空时，事件循环都会先处理所有的微任务队列中的任务，然后再处理一个宏任务。

事件循环的步骤：

  1. 执行全局脚本：加载页面时，浏览器会首先执行全局脚本。
  2. 从事件队列中取出事件：执行完全局脚本后，事件循环会从事件队列中取出一个事件。
  3. 执行事件对应的回调：事件循环将取出的事件对应的回调函数压入调用栈，并执行之。
  4. 清空微任务队列：在当前宏任务（包括其同步代码和异步回调）执行完后，事件循环会处理所有的微任务。在每个微任务执行后，会再次检查微任务队列，直到队列为空。
  5. 渲染UI（如果有必要）：在适当的时机，浏览器会更新渲染UI。
  6. 重复上述步骤：继续从事件队列中取出下一个事件，循环继续。

注意点：

  • 宏任务与微任务：宏任务包括setTimeout、setInterval、I/O、UI事件等；微任务主要是Promise回调和MutationObserver。微任务的执行优先级高于宏任务。
  • 渲染操作的时机：浏览器会在宏任务和微任务之间，以及必要时进行渲染操作。
  • 阻塞现象：虽然JavaScript是单线程的，但长时间运行的脚本可以阻塞事件循环，导致页面无响应。这就是为什么推荐使用异步编程模式的原因之一。

浏览器的事件循环机制是理解异步编程和性能优化的关键。正确地利用事件循环，可以提高应用的响应速度和用户体验。

3、浏览器输入url之后发生了什么

1. 解析URL

  • 浏览器首先解析URL以确定目标地址和协议（如HTTP或HTTPS）。

2. DNS查询

  • 如果URL的域名不在浏览器缓存中，浏览器会发起一个DNS（域名系统）查询以将域名解析为IP地址。

3. 建立连接

  • 浏览器使用解析到的IP地址尝试与服务器建立连接。对于HTTPS，这还包括TLS握手过程来建立安全连接。

4. 发送HTTP请求

  • 一旦建立了连接，浏览器会构建并发送一个HTTP请求给服务器，请求网页的内容。

5. 服务器处理请求并响应

  • 服务器接收到请求后，会处理该请求，并生成一个HTTP响应，将请求的网页数据发送回浏览器。

6. 浏览器接收响应

  • 浏览器接收到服务器的响应数据后，开始解析HTML内容。

7. 渲染页面

  • 浏览器首先构建DOM（文档对象模型）树。
  • 然后，它会构建CSSOM（CSS对象模型）树。
  • 浏览器将DOM和CSSOM合并成一个渲染树。
  • 根据渲染树，浏览器计算所有元素的布局（即它们在页面上的确切位置和大小）。
  • 最后，浏览器绘制页面上的所有元素。

8. 运行JavaScript

  • 如果页面包含JavaScript，浏览器还会解析和执行JavaScript代码，这可能会修改DOM和触发页面的重排和重绘。

9. 加载其他资源

  • 页面中可能还包含了额外的资源，如图片、视频、CSS文件、JavaScript文件等。这些资源的加载可能是同步或异步的，可能会进一步影响页面的渲染过程。

10. 完成加载

  • 一旦所有的资源都加载完成，浏览器会触发load事件，此时页面完全加载完毕。

这个过程涉及到了网络层面的交互、服务器处理、浏览器渲染引擎的工作等多个复杂步骤，每一步都有更深层次的细节和潜在的优化点。
